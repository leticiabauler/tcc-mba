
# Diretrizes Gerais

### Comunicação com o Cliente

* Liderada pelo PO;
* Reuniões mensais de demonstração;
* Base para detalhamento das histórias.

### Estimativa de Tempo

* Avaliação macro do escopo;
* Apoio de mapeamento de processos (ex.: Six Sigma);
* Execução via método ágil.

### Início de Projeto

* Aprovação formal;
* Escopo e documentação definidos;
* *Key users* disponíveis.

### Alterações de Escopo

* Avaliação de impacto;
* Redefinição de prioridades com o cliente;
* Objetivo final preservado.

### Code Review

* Garantia de boas práticas;
* Apoio cruzado em projetos individuais.

### Lições Aprendidas

* Registro em SharePoint;
* Uso incentivado;
* Consulta antes de novas decisões.

### Gestão de Problemas

* Registro formal;
* Discussão no PO Sync;
* Planos de ação obrigatórios.

### Ferramentas

* Ferramentas técnicas registradas;
* Jira como plataforma principal;
* Base para avaliações de qualidade.

### Templates

* Templates para todas as fases;
* Repositório centralizado;
* Padronização como objetivo principal.
